# КолорПро — генератор цветовой схемы по изображению

**КолорПро** — это клиентское веб-приложение (HTML/JS/CSS), которое извлекает доминирующую палитру из загруженного логотипа или любого изображения и генерирует готовую тему оформления: пример интерфейса (кнопка, карточка, акцент), а также код для Tailwind Config и CSS-переменных.  

**Ключевая особенность:** все вычисления происходят локально в браузере, изображения никуда не отправляются.

## Структура проекта
КолорПро/
├── index.html # Основная разметка, стили (Tailwind + кастомные CSS), подключение скриптов
├── colorExtractor.js # Функции для работы с цветом и извлечения палитры из изображения
├── uiManager.js # Класс UIManager – отрисовка палитры, пример интерфейса, генерация кода
├── main.js # Точка входа: обработка загрузки файла, drag&drop, связь UI с логикой
└── README.md # Документация (этот файл)


### Файлы и их назначение

#### 1. `index.html`
- Содержит всю HTML‑разметку (шапка, зона загрузки, превью, палитра, демо‑карточка, вкладки кода).
- Включает Tailwind CSS и ColorThief (сторонние библиотеки через CDN).
- Подключает скрипты в правильном порядке: `colorExtractor.js` → `uiManager.js` → `main.js`.
- Стилизован с учётом адаптивности, содержит анимации (toast, fade-up, drag-over).

#### 2. `colorExtractor.js`
**Глобальный объект:** `window.ColorExtractor`

**Что делает:**
- Преобразование цветов: `rgbToHex`, `hexToRgb`, `rgbToHue`, `getSaturationAndValue`.
- Автоматический выбор ключевых цветов темы: `autoSelectColors(hexColorsArray)` → `{ primary, secondary, accent, text }`.
- Извлечение палитры из файла изображения: `extractPaletteFromFile(file)` → возвращает Promise с массивом hex‑цветов (до 12 цветов).

**Алгоритм извлечения:**
1. Изображение ресайзится до 400px (по большей стороне) для производительности.
2. Усиливается насыщенность (factor 1.2), чтобы лучше выделить малые акценты.
3. Используется библиотека **ColorThief** (метод `getPalette`) для получения до 12 доминирующих цветов.
4. Цвета конвертируются в hex.

**Выбор primary / secondary / accent / text:**
- **primary:** самый насыщенный цвет (при близкой насыщенности — более яркий).
- **secondary:** второй по насыщенности, не равный primary.
- **accent:** среди насыщенных (saturation > 0.3) выбирается цвет с максимальным отличием по тону (hue) от primary.
- **text:** самый тёмный цвет из палитры (если он светлее 0.5 value – используется fallback `#1e1b4b`).

#### 3. `uiManager.js`
**Глобальный класс:** `UIManager`

**Отвечает за:**
- Рендер палитры в DOM (`renderPalette`).
- Обновление демо‑интерфейса (кнопка, бейдж, текст, градиент).
- Генерацию Tailwind config и CSS переменных.
- Переключение между вкладками кода (Tailwind / CSS).
- Управление активным акцентным цветом.

**Основные методы:**
| Метод | Описание |
|-------|----------|
| `setColors(primary, secondary, accent, text)` | Устанавливает все ключевые цвета и обновляет интерфейс/код. |
| `setAccent(accent)` | Меняет только акцентный цвет. |
| `renderPalette(hexArray, activeAccent, onAccentChange, onCopyHex)` | Отрисовывает цветовые карточки с колбэками на клик по цвету (смена акцента) и по hex (копирование). |
| `setActiveAccentClass(activeHex)` | Подсвечивает выбранный акцент в палитре. |
| `updateUIAndCode()` | Внутренний метод, вызываемый при любом изменении цветов. |

**Генерируемый код:**
- **Tailwind config:** расширяет тему `colors` полями `primary`, `secondary`, `accent`, `background`, `surface`, `text`.
- **CSS variables:** определяет `--color-primary`, `--color-accent`, `color-text` и т.д., плюс примеры классов `.btn-primary` и `.card`.

#### 4. `main.js`
**Точка входа** – код обёрнут в IIFE, не использует модули.

**Задачи:**
- Инициализация DOM‑элементов.
- Настройка drag & drop и клика по зоне загрузки.
- Обработка выбора файла (`handleFileSelection`): валидация (размер до 10 МБ, тип image), отображение превью, активация кнопки «Извлечь».
- Функция `generatePaletteAndTheme`:
  - Вызывает `ColorExtractor.extractPaletteFromFile`.
  - Автоматически выбирает ключевые цвета через `autoSelectColors`.
  - Передаёт цвета в `UIManager`.
  - Рендерит палитру с колбэками (смена акцента, копирование hex).
  - Показывает блок с результатами.
- Функция `clearAll` – сброс состояния, скрытие результата.
- Копирование кода по кнопке.

## Запуск и использование

### Требования
- Современный браузер (Chrome, Firefox, Safari, Edge) с поддержкой ES6 и Canvas.
- Локальное хранилище не требуется — всё работает из файловой системы.

### Как открыть
1. Сохраните все четыре файла в **одну папку** (имена как выше).
2. Дважды щёлкните по `index.html` или перетащите его в окно браузера.
3. Страница откроется, будет доступен весь функционал.

> **Важно:** при открытии через `file://` протокол браузер может выдать предупреждения о Cookie или CORS (это нормально, работа приложения не нарушается). Мы отказались от ES‑модулей, чтобы избежать ошибок CORS при локальном открытии.

### Пошаговый сценарий
1. **Загрузите изображение** – кликните по области «Перетащите файл сюда» или перетащите файл (PNG, JPEG, WEBP). Сразу появится превью.
2. **Нажмите «✨ Извлечь палитру и тему»** – через секунду отобразятся:
   - Палитра из 6–12 цветов (каждый цвет можно кликнуть – он станет акцентным; нажатие на hex‑код копирует его).
   - Пример интерфейса (карточка) с обновлёнными цветами.
   - Tailwind config и CSS переменные.
3. **Переключайте вкладки** «Tailwind config» / «CSS variables» – код меняется динамически.
4. **Копируйте код** кнопкой «📋 Скопировать код».
5. **Очистите всё** кнопкой «🗑️ Очистить», чтобы загрузить новое изображение.

## Принципы работы

- **Локальность:** всё обрабатывается в браузере через Canvas и ColorThief.
- **Отзывчивость:** асинхронный анализ не блокирует UI.
- **Кастомизация:** пользователь в любой момент может переопределить акцентный цвет кликом по палитре – пример и код обновятся мгновенно.
- **Технологии:** Vanilla JS, Tailwind (только для стилей макета), ColorThief 2.3.0.
