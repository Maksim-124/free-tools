### Пояснение к папке `tools/`

- **`ColorПРО/`** – генератор цветовой схемы по загруженному изображению (логотип, фото).  
  Работает полностью локально, без отправки данных на сервер.  
  Извлекает доминирующие цвета, строит палитру, демонстрирует готовую тему интерфейса и генерирует код Tailwind / CSS.  
  → Подробнее см. `tools/ColorПРО/README.md`.
- **`FaviconПРО.html`** – инструмент для создания фавиконов.
- **`ТекстПРО.html`** – текстовый комбайн: очистка, дедупликация, извлечение данных (email, ссылки, телефоны).

Остальные файлы в папке `tools/` (если появятся) – аналогичные локальные утилиты, каждая со своей логикой.

## Технологический стек

| Категория     | Технология                       | Назначение                                      |
|---------------|----------------------------------|-------------------------------------------------|
| Язык          | Vanilla JavaScript (ES6+)        | Логика приложения без фреймворков               |

| Деплой        | GitHub Pages                     | Статический хостинг                             |
| Данные        | JSON (tools.json)                | Единый источник истины для каталога             |

## Архитектура и поток данных

1. Загрузка страницы  
   ↓  
2. `DOMContentLoaded` → `initTheme()` + `loadTools()`  
   ↓  
3. `fetch('./tools.json')` → `normalizeData()` → `toolsData[]`  
   ↓  
4. `initFuse(toolsData)` → индексация для поиска  
   ↓  
5. `setActiveCategory('all')` → `applyFilters()` → `renderTools()`  
   ↓  
6. События: поиск / клик по категории / клик по карточке → обновление UI  

### Пояснение потока

- **initTheme** – применяет сохранённую тему (светлая/тёмная) из localStorage.  
- **normalizeData** – убирает лишние пробелы в ключах JSON (например, `"id "` → `"id"`).  
- **initFuse** – настраивает нечёткий поиск с весами по полям `title`, `searchTags`, `description`.  
- **setActiveCategory** – устанавливает текущую категорию, обновляет активный класс на кнопках фильтра и вызывает `applyFilters`.  
- **applyFilters** – комбинирует фильтр по категории и поисковый запрос (через Fuse) и запускает `renderTools`.  
- **renderTools** – очищает контейнер и отрисовывает карточки через `renderCard()`.  
- **debounce** – применяется к полю поиска (задержка 200 мс).  

### Особенности страниц в `tools/`

Страницы-инструменты (например, `ColorПРО/index.html`) – полностью независимые одностраничные приложения.  
Каждая имеет собственную разметку, стили и JS-логику (иногда разделённую на модули).  
Они работают локально (через Canvas, ColorThief и т.д.), не отправляют данные на сервер и могут быть открыты как напрямую из файловой системы, так и через веб-сервер.

## Ключевые функции scripts.js

| Функция            | Ответственность                                                   |
|--------------------|-------------------------------------------------------------------|
| `initTheme()`      | Переключение светлой/тёмной темы с сохранением в localStorage      |
| `normalizeData()`  | Исправление ключей JSON с лишними пробелами                        |
| `initFuse()`       | Настройка нечёткого поиска с весами по полям                       |
| `renderCard()`     | Генерация HTML карточки инструмента                                |
| `renderTools()`    | Отрисовка массива инструментов в сетке                             |
| `applyFilters()`   | Комбинированная фильтрация: категория + поисковый запрос           |
| `setActiveCategory()` | Установка активной категории и обновление UI                    |
| `performSearch()`  | Обработка ввода в поиске, вызов `applyFilters`                     |
| `showToolDetails()`| Открытие модального окна с подробной информацией                   |
| `debounce()`       | Оптимизация ввода в поиске (200 мс)                                |

## Структура данных tools.json

```json
{
  "id": "number",
  "popular": "boolean",
  "title": "string",
  "shortDescription": "string",
  "description": "string",
  "category": "office | design | ai | development | storage | conversion | marketing | other",
  "available": "yes | no",
  "link": "string (URL)",
  "logo": "string (filename, опционально)",
  "hashtags": ["string"],
  "searchTags": ["string"],
  "useCases": ["string"],
  "features": ["string"],
  "freeFeatures": "string",
  "paidFeatures": "string"
}

Описание полей
Поле	Описание
id	Уникальный числовой идентификатор
popular	Если true, на карточке показывается иконка 🔥
title	Название инструмента
shortDescription	Краткое описание (на карточке)
description	Полное описание (в модальном окне)
category	Одна из восьми категорий (см. выше)
available	Доступность в РФ (yes или no)
link	Прямая ссылка на сервис (или относительный путь для локальных инструментов)
logo	Имя файла в папке /tools/logos/ (опционально)
hashtags	Теги для отображения (автоматически фильтруются от служебных)
searchTags	Дополнительные ключевые слова для поиска
useCases	Сценарии использования
features	Список возможностей
freeFeatures	Что входит в бесплатный тариф
paidFeatures	Платные опции
